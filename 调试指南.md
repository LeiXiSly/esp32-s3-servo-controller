# ESP32-C3 舵机控制器 - 调试指南

## 📦 所需材料

| 物品 | 数量 | 备注 |
|------|------|------|
| ESP32-C3 开发板 | 1 块 | 支持 WiFi |
| SG90 舵机 | 1 个 | 9g 微型舵机 |
| 杜邦线 | 若干 | 公对母或母对母 |
| 5V 电源 (可选) | 1 个 | 如果舵机抖动建议外接 |

## 🔧 接线步骤

### 1. SG90 舵机线序识别
- **棕色线**: GND (接地)
- **红色线**: VCC (电源正极 4.8-6V)
- **橙色线**: Signal (信号线)

### 2. ESP32-C3 连接
```
SG90 棕色线 → ESP32-C3 GND
SG90 红色线 → ESP32-C3 5V (或 VIN)
SG90 橙色线 → ESP32-C3 GPIO2
```

### 3. 接线检查清单
- [ ] 确认 GND 共地
- [ ] 确认 VCC 接 5V 引脚 (不要接 3.3V)
- [ ] 确认信号线接 GPIO2
- [ ] 确认接线牢固无松动

## 💻 程序烧录

### 方法一：Arduino IDE

1. **安装 ESP32 开发板支持**
   - 打开 Arduino IDE
   - 文件 → 首选项 → 附加开发板管理器网址
   - 添加：`https://espressif.github.io/arduino-esp32/package_esp32_dev_index.json`
   - 工具 → 开发板 → 开发板管理器
   - 搜索 "esp32" 并安装 "esp32 by Espressif Systems"

2. **选择开发板**
   - 工具 → 开发板 → ESP32 Arduino → **Xiao ESP32C3** (或你使用的具体型号)
   - 工具 → Port → 选择对应的 COM 端口

3. **安装 Servo 库**
   - 项目 → 加载库 → 管理库
   - 搜索 "Servo" 并安装 `ESP32Servo` by Kevin Harrington

4. **修改 WiFi 配置**
   打开 `servo_controller.ino`，修改以下两行：
   ```cpp
   const char* ssid = "YOUR_WIFI_SSID";        // 改成你的 WiFi 名称
   const char* password = "YOUR_WIFI_PASSWORD"; // 改成你的 WiFi 密码
   ```

5. **上传程序**
   - 点击上传按钮 (→)
   - 等待编译和上传完成
   - 打开串口监视器 (波特率 115200)

### 方法二：PlatformIO (VS Code)

1. 安装 PlatformIO 扩展
2. 创建新项目，选择 ESP32-C3
3. 在 `platformio.ini` 添加：
   ```ini
   lib_deps = 
       ESP32Servo
   ```
4. 将代码复制到 `src/main.cpp`
5. 修改 WiFi 配置
6. 点击上传

## 🔍 调试步骤

### 步骤 1: 串口监视器检查

上传程序后，打开串口监视器 (115200 波特率)，应该看到：

```
ESP32-C3 舵机控制器启动
正在连接 WiFi...
WiFi 连接成功！
IP 地址：192.168.1.100
信号强度 (RSSI): -65 dBm
mDNS 启动：http://esp32servo.local
HTTP 服务器已启动
=================================
```

**常见问题：**

| 现象 | 可能原因 | 解决方案 |
|------|----------|----------|
| 一直显示"正在连接 WiFi..." | WiFi 密码错误或信号弱 | 检查密码，靠近路由器 |
| 显示"WiFi 连接失败" | SSID 或密码错误 | 仔细检查配置 |
| 舵机抖动或不转 | 供电不足 | 外接 5V 电源 |
| 无任何输出 | 串口波特率错误或上传失败 | 确认 115200，重新上传 |

### 步骤 2: Web 界面访问

1. **通过 IP 访问**
   - 在浏览器输入串口显示的 IP 地址
   - 例如：`http://192.168.1.100`

2. **通过 mDNS 访问** (如果支持)
   - 在浏览器输入：`http://esp32servo.local`
   - 注意：部分 Windows 系统需要安装 Bonjour 服务

3. **手机访问**
   - 确保手机和 ESP32 在同一 WiFi
   - 用手机浏览器访问 IP 地址

### 步骤 3: 舵机测试

1. **基础测试**
   - 打开 Web 界面
   - 拖动角度滑块
   - 观察舵机是否平滑转动
   - 检查 Web 界面的舵机臂动画是否同步

2. **速度测试**
   - 点击不同速度按钮 (慢/中/快/最快)
   - 拖动角度滑块
   - 观察舵机转动速度变化

3. **预设位置测试**
   - 点击 0°、90°、180° 按钮
   - 确认舵机准确到达指定位置

## ⚠️ 常见问题与解决方案

### 问题 1: 舵机抖动严重

**原因**: 供电不足

**解决方案**:
```
方案 A: 使用外接 5V 电源
- 外接 5V 电源正极 → SG90 红色线
- 外接 5V 电源负极 → ESP32-C3 GND (共地)
- ESP32-C3 5V 引脚不接

方案 B: 使用更强力的 USB 电源
- 使用 2A 以上的 USB 充电器
- 避免使用电脑 USB 口供电
```

### 问题 2: 舵机转动角度不准确

**原因**: PWM 信号问题或舵机机械限位

**解决方案**:
1. 检查舵机是否质量良好 (SG90 有一定误差是正常的)
2. 在代码中微调角度映射
3. 避免设置超出舵机物理范围的角度

### 问题 3: Web 界面加载慢或卡顿

**原因**: WiFi 信号弱或服务器负载高

**解决方案**:
1. 检查 WiFi 信号强度 (RSSI)
   - RSSI > -70 dBm: 良好
   - RSSI < -80 dBm: 信号弱，靠近路由器
2. 减少同时访问的设备数量
3. 重启 ESP32

### 问题 4: 舵机无法到达 0°或 180°

**原因**: SG90 物理限位或 PWM 脉宽问题

**解决方案**:
```cpp
// 在代码的 setup() 中添加：
myServo.setPeriodHertz(50);    // 标准 50Hz
myServo.attach(SERVO_PIN, 500, 2400); // 调整 PWM 脉宽范围
```

### 问题 5: 上传失败

**可能原因**:
- 开发板型号选择错误
- COM 端口错误
- 缺少驱动

**解决方案**:
1. 确认开发板型号正确
2. 检查设备管理器中的 COM 端口
3. 安装 CH340 或 CP2102 驱动 (根据开发板芯片)
4. 按住 BOOT 键再按 RESET 键进入下载模式

## 📱 使用技巧

1. **快速定位**: 直接点击 0°/90°/180° 按钮快速归位
2. **精细调节**: 拖动滑块进行微调
3. **速度选择**: 
   - 慢速 (1-2): 精细操作
   - 中速 (3-5): 日常使用
   - 快速 (6-8): 快速响应
   - 最快速 (9-10): 演示效果

## 🔐 安全注意事项

1. **不要长时间堵转**: 舵机到达限位后不要持续发送同角度信号
2. **避免频繁快速转动**: 会缩短舵机寿命
3. **注意供电**: 确保电源稳定，避免电压波动
4. **散热**: 长时间使用时注意 ESP32 和舵机的温度
